# Asymmetric Distillation & Drafting (ADD) Implementation Guide

**Context for the AI Agent (e.g., Gemini Flash):**
The overarching goal is to implement the ADD architecture to optimize the Hexamind system for a heavily constrained local environment (Dual-Core Xeon CPU, 42GB RAM). Autoregressive generation on the 14B model is a severe bottleneck. The solution is two-fold:
1. **Algorithmic Context Pruning**: Shrink the context window before LLM ingestion using non-LLM Python libraries. 
2. **Speculative Agent Drafting**: Use a 0.5B tiny model to generate the heavy text, and use the 7B/14B models ONLY as JSON-based "diff" editors.

---

## Phase 1: Algorithmic Context Shrinking

**Goal:** Intercept the raw Web Search results before they are handed to the LLM and aggressively strip grammatical "fat".

**Files to Modify:**
- `ai-service/research.py` 

**Implementation Details:**
1. Install a lightweight NLP payload (such as `nltk` removing stopwords, or `spacy` core-web-sm if feasible). 
2. Locate the methods that hydrate candidate text, specifically `_extract_evidence_excerpt()` and `_fetch_page_text()`.
3. Create a new utility function `_prune_context_to_triplets(text: str) -> str`.
4. *Logic:* 
   - Tokenize the text.
   - Remove all Stop Words (the, a, is, was, etc.), generic adjectives, and adverbs.
   - Retain ONLY Named Entities, Nouns, Verbs, and Numbers.
   - Return the text as dense, comma-separated keywords or crude relationships.
5. Apply this function so that the returned `ResearchContext` is 50% smaller in token size but retains all empirical data.

---

## Phase 2: The "Fast Drafter" Agent 

**Goal:** Remove text generation from heavy models entirely. 

**Files to Modify:**
- `ai-service/agent_model_config.py`
- `ai-service/worker_agents.py`

**Implementation Details:**
1. In `agent_model_config.py`, introduce a new agent: `drafter`. Map its `primary_ollama_model` to a highly lightweight model like `qwen2.5:0.5b` or `qwen:0.5b`. Set its `max_tokens` to `1000`.
2. In `worker_agents.py`, create a `DraftingWorker`. 
   - Its prompt should take the pruned `ResearchContext` and write a full, cohesive, lengthy Markdown draft.
   - It needs to run *first*, serving as the foundation for all other agents.

---

## Phase 3: Converting Experts into "Editors"

**Goal:** The Historian, Auditor, Analyst, and Researcher must NEVER output paragraphs again.

**Files to Modify:**
- `ai-service/worker_agents.py`
- `ai-service/reasoning_graph.py`

**Implementation Details:**
1. Rework the Prompts for the "Diamond Experts" (`historian`, `auditor`, etc.).
2. You must pass them the `Draft` generated by the 0.5B model, NOT just the raw context.
3. Radically alter their system prompt: 
   > "You are an Editor. Review the Draft against the evidence. DO NOT rewrite the draft. Output ONLY a valid JSON array of corrections in the format: `[{"original_text_snippet": "...", "replacement_text": "..."}]`. If the draft is perfect, return an empty array `[]`."
4. Cap their `max_tokens` at a tiny ceiling (e.g., `150` tokens) to mechanically ensure they cannot generate paragraphs.

---

## Phase 4: The Assembler

**Goal:** Merge the heavy AI's JSON edits onto the tiny AI's text draft autonomously.

**Files to Modify:**
- `ai-service/reasoning_graph.py`

**Implementation Details:**
1. After the Diamond Experts return their JSON arrays, run a pure python function `apply_diffs_to_draft(initial_draft: str, diffs: list) -> str`.
2. This function simply executes a string `replace()` using the `original_text_snippet` against the `initial_draft`.
3. Pass the resulting heavily-edited, highly-accurate draft straight to the final output, skipping the `Synthesizer` generation or minimizing the Synthesizer's role to just basic formatting.

---

## Testing Protocol
1. Run `python ai-service/run_demo.py "impact of remote work on urban development"`.
2. **Success Criteria:** 
   - The Drafter (0.5B) should execute the heaviest text generation quickly.
   - The Diamond Experts (7B/14B) should ONLY output raw JSON.
   - The final output is assembled correctly.
3. Once the test passes and the architecture is verified stable, **DELETE** this guide file (`ADD_Implementation_Guide.md`) to keep the workspace clean.
